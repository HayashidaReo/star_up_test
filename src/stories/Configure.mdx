# Star Up Test - Storybook

このStorybookでは、Star Up Testアプリケーションのコンポーネントを**Atomic Design**の原則に従って構成しています。

## Atomic Design について

Atomic Designは、ユーザーインターフェースを5つの階層で構成する設計手法です：

### ⚛️ Atoms (原子)
最も基本的なUI要素。単体で機能する最小単位のコンポーネント
- Button, Input, Badge, Avatar など
- 他のコンポーネントに依存しない
- 再利用性が非常に高い

### 🧬 Molecules (分子)
複数のAtomsを組み合わせた、特定の機能を持つコンポーネント
- CurrencySelect, ParticipantItem, ExpenseRow など
- 単一の責務を持つ
- 特定のコンテキストで意味を持つ

### 🦠 Organisms (有機体)
MoleculesやAtomsを組み合わせた、複雑な機能を持つコンポーネント
- ParticipantsSection, ExpensesSection, SettlementSection など
- 複数の責務を持つことがある
- ページの一部分を構成する

### 📄 Pages (ページ)
実際のページを表すコンポーネント
- Home など
- Organismsを組み合わせて完全なUIを構成
- ルーティングやデータフェッチを含む

## 技術スタック

- **Next.js 14** (App Router)
- **TypeScript** - 型安全性の確保
- **Tailwind CSS** - ユーティリティファーストのスタイリング
- **Shadcn/ui** - 再利用可能なUIコンポーネント
- **Zustand** - シンプルな状態管理
- **Zod** - スキーマバリデーション

## 設計原則

- **シンプルさと表現力**: 少ない記述量で直感的に扱えるコード
- **型安全性の徹底**: 実行時エラーを未然に防ぐ型システム  
- **開発者体験と所有権**: UIコードをプロジェクト自身が所有し、自由にカスタマイズ
- **コンポーネントの単一責任**: 各コンポーネントは1つの明確な責務のみを持つ
